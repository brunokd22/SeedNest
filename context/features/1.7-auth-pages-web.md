# Spec 1.7 — Auth Pages (Web)

## Goal
Build all five authentication pages inside `apps/web/src/app/(auth)/` using shadcn/ui, react-hook-form, Zod, and the better-auth client.

**Depends on:** Spec 1.6

---

## Shared Design System
- Container: centred card, `max-w-md`
- Header: "SeedNest" logo text at top of each card
- Card: white background, `shadow-sm`, `rounded-lg`, `p-8`
- Primary button colour: `#2D6A4F`
- Toast notifications via **sonner**
- All field errors shown inline via react-hook-form

---

## Pages

### 1. `sign-up/page.tsx`

**Fields:**
| Field | Validation |
|---|---|
| Full Name | min 2 chars |
| Email | valid email |
| Password | min 8 chars, show/hide toggle |
| Role | `<Select>` with options: "Manager" / "Customer" |

**On submit:**
```ts
authClient.signUp.email({ name, email, password, role })
```

**On success:** redirect to `/verify-email?email=<encoded_email>` + toast "Check your email!"

**Bottom link:** "Already have an account? Sign in" → `/sign-in`

---

### 2. `sign-in/page.tsx`

**Fields:**
| Field | Validation |
|---|---|
| Email | valid email |
| Password | required |

**On submit:**
```ts
authClient.signIn.email({ email, password })
```

**On success:** read session role, redirect:
- `MANAGER` → `/dashboard`
- `CUSTOMER` → `/explore`

**On failure:** show inline error "Invalid credentials"

**Additional UI:**
- "Forgot password?" link → `/forgot-password`
- "Create an account" link → `/sign-up`
- "Continue with Google" button:
  ```ts
  authClient.signIn.social({ provider: 'google' })
  ```

---

### 3. `verify-email/page.tsx`

**URL param:** `?email=<email>` — display the email address in the UI.

**OTP Input:**
- 6 separate single-character `<Input>` elements
- Auto-focus next input on entry
- Auto-focus previous input on backspace

**On submit:**
```ts
authClient.emailOtp.verifyEmail({ email, otp })
```

**On success:** read session role, redirect to `/dashboard` or `/explore`.

**Resend button:**
- On click: resend verification email
- Show 60-second cooldown countdown after clicking

---

### 4. `forgot-password/page.tsx`

**Fields:**
| Field | Validation |
|---|---|
| Email | valid email |

**On submit:**
```ts
authClient.forgetPassword({ email, redirectTo: '/reset-password' })
```

**On success (always):** show message "If that email exists, you'll receive a reset link." — never confirm whether the email exists.

---

### 5. `reset-password/page.tsx`

**URL param:** `?token=<token>` — read token from query string.

**Fields:**
| Field | Validation |
|---|---|
| New Password | min 8 chars |
| Confirm Password | must match New Password |

**On submit:**
```ts
authClient.resetPassword({ token, newPassword })
```

**On success:** redirect to `/sign-in` + toast "Password reset successfully"

---

## Notes
- Each page is a standalone `'use client'` component — no additional layout wrappers needed beyond `(auth)/layout.tsx`
- Use `zodResolver` from `@hookform/resolvers/zod` with schemas from `@seednest/shared`
- All `authClient` calls should be wrapped in `try/catch` to surface errors cleanly

---

## Completion Criteria
- [ ] Sign-up creates a user and triggers verification email
- [ ] Verify-email OTP input auto-advances between boxes
- [ ] Sign-in redirects to the correct dashboard by role
- [ ] Forgot-password shows success message without revealing email existence
- [ ] Reset-password validates matching passwords before submitting
