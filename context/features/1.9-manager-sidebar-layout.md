# Spec 1.9 — Manager Sidebar Layout (Web)

## Goal
Build the collapsible Manager dashboard layout with sidebar navigation, header bar, and role guard.

**Depends on:** Spec 1.6

---

## Files to Create / Modify

```
apps/web/src/
  app/(manager)/
    layout.tsx                    ← Manager layout shell
  components/layout/
    ManagerSidebar.tsx            ← Sidebar component
    UserMenu.tsx                  ← Avatar + dropdown (shared with customer nav)
```

---

## Layout Behaviour

| Viewport | Sidebar |
|---|---|
| Desktop | Always visible, collapsible between 256px (expanded) and 64px (icon-only) |
| Mobile | Hidden by default; toggled open via a Sheet/drawer |

**Collapse state** is persisted to `localStorage` under the key `'sidebar-collapsed'`.

---

## `ManagerSidebar.tsx`

### Props
```ts
interface ManagerSidebarProps {
  isCollapsed: boolean;
  onToggle: () => void;
}
```

### Sidebar Header
- **Expanded:** "SeedNest" logo text + collapse toggle button (ChevronLeft icon)
- **Collapsed:** collapse toggle button only (ChevronRight icon), centred

### Navigation Items
| Label | Route | Icon (lucide-react) |
|---|---|---|
| Overview | `/dashboard` | `LayoutDashboard` |
| Nurseries | `/dashboard/nurseries` | `TreePine` |
| Orders | `/dashboard/orders` | `ShoppingBag` |
| Issues | `/dashboard/issues` | `MessageSquare` |
| Reports | `/dashboard/reports` | `BarChart3` |
| Notifications | `/dashboard/notifications` | `Bell` + unread count badge |

### Active Route Styling
Use `usePathname()` to detect the active route.  
Active item: `bg-primary/10 text-primary font-medium`  
Inactive item: `text-muted-foreground hover:bg-muted`

### Collapsed State
- Show icon only, centred horizontally
- Wrap each nav item in a shadcn **Tooltip** showing the label on hover

### Transitions
Smooth width transition using CSS: `transition-all duration-200 ease-in-out`

---

## `(manager)/layout.tsx`

### Structure
```
<div className="flex h-screen">
  <ManagerSidebar isCollapsed={isCollapsed} onToggle={toggle} />
  <div className="flex flex-col flex-1 overflow-hidden">
    <header>...</header>
    <main className="flex-1 overflow-y-auto p-6">
      {children}
    </main>
  </div>
</div>
```

### Header Bar
- **Left:** Dynamic page title derived from the current pathname (e.g., `/dashboard/orders` → "Orders")
- **Right:**
  - `NotificationBell` component: Bell icon + red badge showing unread notification count (placeholder count of `0` for now; wired up in a later phase)
  - `UserMenu` component

### Role Guard
```ts
const session = await auth.getSession(); // server-side
if (session?.user.role !== 'MANAGER') redirect('/explore');
```

---

## `UserMenu.tsx`

shadcn `DropdownMenu` triggered by an `Avatar`:
- Avatar shows user's image if available, otherwise user initials
- Dropdown items:
  - Profile (placeholder link)
  - Sign Out → calls `authClient.signOut()` then `router.push('/sign-in')`

---

## Main Content Offset
The `<main>` element should have a left margin matching the sidebar's current width:
- Expanded: `ml-64`
- Collapsed: `ml-16`

Apply the same CSS transition as the sidebar for a smooth animation.

---

## Completion Criteria
- [ ] Sidebar renders on desktop with all 6 nav items
- [ ] Clicking the toggle collapses sidebar to icon-only mode
- [ ] Collapsed state persists across page refreshes (localStorage)
- [ ] Active route item is highlighted
- [ ] Tooltips appear on icon-only nav items when collapsed
- [ ] Mobile: sidebar opens as a Sheet/drawer
- [ ] CUSTOMER visiting `/dashboard` is redirected to `/explore`
- [ ] Sign Out clears session and redirects to `/sign-in`
