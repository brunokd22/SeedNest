## File Specifications

### `src/config/env.ts`
Validate with Zod and export a typed `env` object. **Throw at startup** if any required var is missing.

Required vars:
```
DATABASE_URL
BETTER_AUTH_SECRET
BETTER_AUTH_URL
RESEND_API_KEY
STRIPE_SECRET_KEY
STRIPE_WEBHOOK_SECRET
R2_ACCOUNT_ID
R2_ACCESS_KEY_ID
R2_SECRET_ACCESS_KEY
R2_BUCKET_NAME
R2_PUBLIC_URL
FRONTEND_URL
NODE_ENV
PORT                    ← default: 4000
```

### `src/config/prisma.ts`
Prisma client singleton — import and reuse across the app.

### `src/config/r2.ts`
Initialize Cloudflare R2 / S3 client using `@aws-sdk/client-s3` and `env` vars.

### `src/config/stripe.ts`
Initialize Stripe client with `env.STRIPE_SECRET_KEY`.

### `src/config/resend.ts`
Initialize Resend client with `env.RESEND_API_KEY`. (Email helpers added in Spec 1.5.)

### `src/app.ts`
Express app factory — **no `listen` call here**.
- Apply: `helmet`, `cors` (allow `env.FRONTEND_URL`), `morgan` (dev only)
- Apply: `express.json()`
- Mount: `GET /api/health` → `{ status: "ok", timestamp: new Date().toISOString() }`
- Mount: `/api/auth` → better-auth handler (placeholder comment for now)
- Apply: global error handler last

### `src/index.ts`
- Import `app` from `app.ts`
- Call `app.listen(env.PORT)`
- Log: `Server running at http://localhost:${env.PORT}`

### `src/middleware/errorHandler.ts`
Global Express error handler — catches errors, returns JSON `{ error: message }` with appropriate status code.

### `src/middleware/validate.ts`
Zod request validation wrapper — validates `req.body` / `req.params` / `req.query` against a given schema.

### `src/middleware/auth.ts`
Stubs only — full implementation in Spec 1.5:
- `requireAuth` — placeholder
- `requireRole(...roles)` — placeholder

### `src/utils/haversine.ts`
```ts
export function haversine(lat1: number, lon1: number, lat2: number, lon2: number): number
```
Returns distance in **kilometres** using the haversine formula.

### `src/utils/asyncHandler.ts`
Wraps async Express route handlers to forward errors to `next()`.

### `tsconfig.json`
Extends `packages/tsconfig/base.json`. Paths resolve from `src/`.

### `tsup.config.ts`
Bundles `src/index.ts` → `dist/`.

### `.env.example`
All env var keys listed with empty values.
