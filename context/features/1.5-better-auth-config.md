# Spec 1.5 — Better Auth Configuration

## Goal
Configure Better Auth on the backend, create email templates with React Email, and implement auth middleware.

**Depends on:** Spec 1.2, Spec 1.3

## `src/config/auth.ts`

Configure Better Auth with:
- **Prisma adapter** → use the Prisma client singleton from `config/prisma.ts`
- **Email/password** provider:
  - `requireEmailVerification: true`
  - On verification: call `sendVerificationEmail(email, url)`
- **Google OAuth** (optional, gated on `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` env vars)
- **Session config:**
  - Store `userId` and `role` in session
  - Default role to `CUSTOMER` if not provided on sign-up
- **Additional sign-up fields:** `name` (string), `role` (UserRole)
- **`baseURL`:** `env.BETTER_AUTH_URL`
- **`secret`:** `env.BETTER_AUTH_SECRET`
- **`trustedOrigins`:** `[env.FRONTEND_URL]`

## `src/config/resend.ts`

Extend the Resend singleton (initialised in Spec 1.2) with helper functions. Install `@react-email/render` if not already present.

```ts
// From address for all emails
const FROM = 'SeedNest <noreply@seednest.com>';

export async function sendVerificationEmail(to: string, url: string): Promise<void>
export async function sendPasswordResetEmail(to: string, url: string): Promise<void>
export async function sendWelcomeEmail(to: string, name: string, role: UserRole): Promise<void>
```

Each helper uses `render(<EmailTemplate />)` and calls `resend.emails.send(...)`.

## `src/middleware/auth.ts` (Full Implementation)

Replace the Spec 1.2 stub with:

### TypeScript augmentation
```ts
declare global {
  namespace Express {
    interface Request {
      user?: AuthUser;
    }
  }
}
```

### `requireAuth` middleware
1. Read the session from Better Auth (`auth.api.getSession`)
2. Attach `req.user` from the session
3. Return `401 { error: 'Unauthorized' }` if no valid session

### `requireRole(...roles: UserRole[])` middleware
1. Assert `req.user` exists (call `requireAuth` first in route chain)
2. Return `403 { error: 'Forbidden' }` if `req.user.role` not in `roles`
